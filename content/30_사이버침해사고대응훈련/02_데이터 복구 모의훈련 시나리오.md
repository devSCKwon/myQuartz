## 1. 훈련 개요

### 1.1 훈련 목적

본 모의훈련은 랜섬웨어 감염으로 인한 데이터 손상 상황을 가정하여 (주)세진 전산팀의 데이터 복구 역량을 검증하고, 실제 사고 발생 시 신속하고 정확한 대응이 가능하도록 절차를 숙지시키는 것을 목적으로 한다.

### 1.2 훈련 범위

- **대상 시스템**: ERP 서버, 파일 서버, 업무용 PC
- **대상 데이터**: 업무 중요도에 따른 핵심 데이터
- **참여 인원**: 전산팀 3명
- **훈련 소요시간**: 약 8시간 (09:00 ~ 17:00)

---

## 2. 사고 시나리오

### 2.1 사고 발생 배경

**(주)세진**은 현대자동차의 1차 부품협력사로서, 자동차 부품 설계 및 제조 업무를 수행하고 있다. 2025년 8월 15일 오전 9시 30분, 재경팀 직원이 협력업체로부터 받은 것으로 추정되는 피싱 메일의 첨부파일을 실행하게 된다.

### 2.2 사고 전개 상황

#### 09:30 - 초기 감염

- 재경팀 김○○ 책임매니저가 "견적서.pdf.exe" 파일을 실행
- Locky 랜섬웨어가 해당 PC에 설치되고 네트워크를 통해 확산 시작

#### 09:35 - 감염 확산

- 파일 서버의 공유 폴더에 저장된 문서들이 암호화 시작
- 설계도면, 기술문서, 회계자료 등 핵심 업무 데이터 암호화

#### 09:40 - ERP 서버 감염

- 네트워크를 통해 ERP 데이터베이스 서버까지 감염
- 생산관리, 자재관리, 품질관리 데이터 암호화

#### 09:45 - 사고 인지

- 전산팀이 사용자 신고를 통해 랜섬웨어 감염 사실 인지
- 바탕화면에 랜섬웨어 메시지 확인

### 2.3 피해 현황

- **감염된 시스템**: 
  - ERP 서버 (Windows Server 2019)
  - 파일 서버 (Windows Server 2022)
  - 업무용 PC 10대 (Windows 11)

- **암호화된 데이터**:
  - ERP 데이터베이스 (약 500GB)
  - CATIA 설계도면 파일 (약 200GB)
  - 기술문서 및 매뉴얼 (약 100GB)
  - 회계 및 인사 데이터 (약 50GB)

## 4. 훈련 절차

### 4.1 단계별 훈련 절차

#### 단계 1: 사고 인지 및 초동대응 (목표: 15분)

1. **사고 신고 접수** (09:30)
   - 사용자로부터 시스템 이상 신고 접수
   - 랜섬웨어 감염 증상 확인

2. **초동대응 조치** (09:35)
   - 감염된 시스템을 네트워크에서 즉시 분리
   - 추가 감염 확산 방지를 위한 네트워크 차단
   - 관련 부서 및 경영진 보고

3. **사고 대응팀 구성** (09:40)
   - 전산팀 긴급 소집
   - 역할 분담 및 복구 계획 수립

#### 단계 2: 백업본 식별 및 검증 (목표: 30분)

1. **백업 현황 확인** (09:45)
   - 각 시스템별 최신 백업본 위치 및 상태 확인
   - 백업 일정 및 보관 주기 검토

2. **백업 무결성 검증** (10:00)
   - 백업 데이터의 랜섬웨어 감염 여부 확인
   - 백업 파일 체크섬 검증
   - 복구 가능한 백업본 선정

#### 단계 3: 복구 환경 구축 (목표: 30분)

1. **격리 환경 구성** (10:15)
   - 랜섬웨어 감염으로부터 안전한 독립 네트워크 구축
   - 복구용 임시 서버 준비

2. **복구 도구 준비** (10:30)
   - 백업 소프트웨어 및 복구 도구 준비
   - 시스템 이미지 복구를 위한 부팅 디스크 준비

#### 단계 4: 데이터 복구 실행

1. **ERP 데이터베이스 복구** (10:45 ~ 12:30)
   - 최우선 복구 대상인 ERP DB 복구 시작
   - 2025년 8월 14일 18:00 백업본 사용
   - 복구 진행 상황 실시간 모니터링

2. **파일 서버 복구** (12:30 ~ 15:15)
   - 설계도면 파일 우선 복구
   - 2025년 8월 14일 20:00 백업본 사용
   - 용량 대비 복구 시간 산정

#### 단계 5: 복구 검증 및 테스트 (목표: 90분)

1. **데이터 무결성 확인** (15:15)
   - 복구된 데이터베이스 일관성 검사
   - 파일 무결성 및 접근 권한 확인

2. **기능 테스트** (15:45)
   - ERP 시스템 로그인 및 주요 기능 테스트
   - 파일 서버 접근 및 파일 열기 테스트
   - 백업 시점 이후 데이터 손실 범위 확인

3. **성능 테스트** (16:15)
   - 시스템 성능 및 응답속도 확인
   - 사용자 접속 테스트

### 4.2 예상 이슈 및 대응방안

#### 예상 이슈 1: 백업 데이터 감염

- **상황**: 백업 데이터에서도 랜섬웨어 흔적 발견
- **대응**: 더 이전 백업본 사용, 데이터 손실 범위 확대

#### 예상 이슈 2: 복구 시간 초과
- **상황**: RTO 목표 시간 내 복구 불가능
- **대응**: 우선순위 재조정, 부분 복구 후 순차 진행

#### 예상 이슈 3: 하드웨어 장애
- **상황**: 복구 과정에서 서버 하드웨어 문제 발생
- **대응**: 예비 장비 활용, 클라우드 임시 환경 구축
